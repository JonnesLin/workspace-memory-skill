# Memory Grooming Checklist

Use this checklist when the shared memory starts to feel noisy, repetitive, or expensive to scan.

## `PROJECT_PROGRESS.md`

- Is the top `Current Snapshot` still accurate
- Are the newest entries ordered correctly
- Are there old entries that no longer belong in the default startup context
- Can repeated monitoring entries be compressed into one summary
- Should older entries move into `archive/project-progress/`

## `WORKING_MEMORY.md`

- Are any rules duplicated
- Are any notes actually session logs and not durable memory
- Did any earlier rule become obsolete
- Can a project-specific note be rewritten into a cleaner durable lesson
- Is every remaining note still worth reading in future sessions

## Archive Rules

- Archive progress logs, not durable memory, by default
- Prefer moving old entries only after summarizing what still matters
- Do not force archive creation if the main progress log is still easy to scan
