# Working Memory

This file stores durable, reusable memory for the workspace.

Update this file selectively. It is not a raw session log.

## Stable User Preferences

- Shared memory files should be written in English unless the workspace specifies otherwise.

## Workspace Workflow Rules

- Treat `AGENTS.md` as the session entry point.
- For non-trivial tasks, read `PROJECT_PROGRESS.md` and `WORKING_MEMORY.md` before project-specific reasoning or edits.
- For trivial or context-free tasks, reading the memory files may be skipped.
- Before finishing a non-trivial session, decide whether either memory file should be updated.
- Update `PROJECT_PROGRESS.md` for meaningful state changes, investigations, resolutions, or clarified next steps.
- Update `WORKING_MEMORY.md` only for durable preferences, constraints, habits, or reusable lessons.
- Avoid filler updates that add no future value.

## Reusable Lessons

- Shared progress should preserve prior entries and keep the newest session at the top.
- Shared working memory should be curated and deduplicated.
- Reusable notes should be distilled summaries rather than copied conversation text.
- When a problem is worth remembering, store both the problem and the resolution.

## Project-Specific Durable Notes

- Add long-lived project assumptions here only when they still matter to future sessions.

## Current Scope Assumptions

- This memory system is workspace-wide rather than project-specific unless the workspace defines a different scope.
- Project-specific entries in `PROJECT_PROGRESS.md` should name the affected path clearly.
- Additional per-project memory files are out of scope unless future sessions decide they are necessary.
